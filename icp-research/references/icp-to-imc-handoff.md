# ICP-to-IMC Handoff

How to package ICP research outputs so they integrate cleanly into `mkt-imc` planning.

---

## What IMC Needs from ICP Research

| ICP Output | What IMC Uses It For |
|-----------|---------------------|
| **Persona profiles** | Persona-pillar weight matrix, channel fit analysis |
| **Pain intensity scores** | Pillar prioritization, angle trigger selection |
| **Emotional drivers (ranked)** | Hook type and trigger matching per persona |
| **VoC library (15+ quotes)** | Validation checkpoints, language matching |
| **Decision journey barriers** | Objection-handling angles |
| **Audience Habitat Map (per persona)** | Persona-channel fit using 3-level hierarchy |
| **Competitive gaps** | Differentiation angles |

---

## Handoff Format

Format your ICP outputs using the structure defined in `mkt-imc/references/icp-integration.md`. That file is the canonical format — it contains the exact tables and examples IMC expects for each section:

- Persona Quick Reference table
- Audience Habitat Map (per persona)
- VoC Library with quote categories
- Decision Journey Barriers
- Competitive Gap Format
- Emotional Driver Rankings

---

## Integration Modes

Your output completeness determines which IMC mode activates:

| What You Have | IMC Mode | What It Unlocks |
|--------------|----------|-----------------|
| Full ICP research (personas + VoC + journey + habitats) | **Enhanced** | Persona-specific angles, VoC validation at every step, habitat-informed channels |
| Partial research (some personas or quotes) | **Guided** | Import what you have, fill gaps with targeted questions |
| No research | **Quick ICP** | Lightweight 18-question capture within IMC |

---

## Quality Checklist Before Handoff

- [ ] At least 2 personas with pain intensity scores
- [ ] 15+ VoC quotes with attribution
- [ ] Emotional drivers ranked per persona
- [ ] Habitat map with 3-5 entries per persona (covering discovery, trust, conversion roles)
- [ ] Decision journey barriers identified
- [ ] Competitive gaps documented
